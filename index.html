Engineering Mindset

How I Think

01

Approaching New Problems

Break the problem into the smallest unit, build outward.

  • Define input, output, and constraints first
  • Sketch data flow before writing code
  • Build dumbest working version first
  • Refactor only after it works correctly
02

Debugging Methodology

Narrow down systematically — reproduce, isolate, fix.

  • Reproduce bug with minimal test case
  • Add logs at boundaries, not everywhere
  • Binary search through codebase mentally
  • Read the error message fully before Googling
03

Designing APIs & Systems

Think about the consumer before the implementation.

  • Design from the client's perspective first
  • Use consistent naming and error formats
  • Validate input strictly at the boundary
  • Version from day one
04

Learning New Technology

Learn by building something real, not just tutorials.

  • Read the official docs, not just Stack Overflow
  • Build a tiny working project in the first hour
  • Break it intentionally to understand limits
  • Teach it back to solidify understanding

Security Mindset

Security in Practice

SQL Injection Prevention BLOCKED
# ❌ Vulnerable
query = f"SELECT * FROM users WHERE name='{user_input}'"

# ✅ Parameterized Query
cursor.execute("SELECT * FROM users WHERE name=%s", (user_input,))
Attack: ' OR 1=1 -- → BLOCKED by parameterization
Rate Limiting PROTECTED
# Flask-Limiter
@app.route("/api/login")
@limiter.limit("5 per minute")
def login(): ...
Brute force limited to 5 attempts/minute per IP
XSS Prevention BLOCKED
# ❌ Vulnerable
element.innerHTML = userContent

# ✅ Safe
element.textContent = userContent
<script>steal()</script> → Rendered as text, not executed
JWT Validation VALIDATED
# JWT with expiry
payload = jwt.decode(token, SECRET_KEY,
algorithms=["HS256"])
check_expiry(payload["exp"])
Signature verified + expiry checked. Tampered tokens rejected.
